CGI(Common Gateway Interface) 프로그램
 - 사용자가 직접 아이콘을 더블클릭하거나 명령 창(또는 터미널)을 통해 실행시키는
 프로그램을 일반적으로 '애프리케이션' 또는 '데스크톱 애플리케이션'이라고 합니다.
 반면에 사용자가 웹 서버를 통해 간접적으로 실행시키는 프로그램이 '웹 애플리케이션' 입니다.
서블릿(Servlet)
 - 자바 CGI 프로그램은 C/C++처럼 컴파일 방식입니다. 자바로 만든 CGI 프로그램을 '서블릿(Servlet)'이라고
 부릅니다. 자바 서블릿이 다른 CGI 프로그램과 다른 점은, 웹 서버와 직접 데이터를 주고밧디 않으면, 전문
 프로그램에 의해 관리된다는 것입니다.
 * 서블릿 컨테이너
  - 서블릿의 생성과 실행, 소멸 등 생명주기를 관리하는 프로그램을 '서블릿 컨테이너(Servlet Container)'
  라 합니다. 서블릿 컨테이너가 서블릿을 대신하여 CGI 규칙에 따라 웹 서버와 데이터를주고받습니다. 따라서
  서브릿 개발자는 더이상 CGI 규칙에 대해 알 필요가 없습니다. 대신 서블릿 컨테이너와
  서블릿 사이의 규칙을 알아야 합니다.

          요청 위임                     실행
        ----------->               ----------->
  웹서버   CGI규칙    서블릿 컨테이너 (Selvelt규칙) 서블릿(.class)
        <-----------               <-----------
          결과 리턴                     결과
자바 웹 애플리케이션 개발자는 JavaEE 기술 사양에 포함된 Servlet 규칙에 따라 CGI 프로그램을 만들고
배포합니다.

정리: 서블릿 컨테이너는 서블릿의 생성에서 실행, 소멸까지 서블릿의 생명주기를
관리하는 프로그램입니다. 클라이언트부터 요청이 들어오면, 서블릿 컨테이너는 호출
규칙에 따라 서블릿의 메서드를 호추합니다. 서블릿 호출 규칙은 javax.servlet.Servlet
인터페이스에 정의되어 있습니다. 따라서 서블릿을 만들 때는 반드시 Servlet
인터페이스를 구현해야만 합니다.

Servelt이라는 규칙 외에 JSP를 만들고 실행하는 규칙, EJB(Enterprise JavaBeans)
라는 분산 컴포넌트에 관한 규칙, 웹 서비스 관한 규칙 등 기업용 애프리케이션에 제작한
필요한 기술들의 사양을 정의한 것을 Java EE라고 합니다. Java EE는 기능 확장이 쉽고,
이기종 간의 이식이 쉬우며, 신뢰성과 보안성이 높고, 트랜잭션 관리와
분산 기능을 쉽게 구현할 수 있는 기술을 제공합니다.

Java EE  사양에 따라 개발된 서버를 'Java EE 구현체(Implementation)' 또는
'Was(Web Application Server)' 라고 부릅니다. 상용 제품으로는 JEUS(티맥스소프트),
WEbLogic(오라클), WebSphere(IBM), JBoss Enterprise(레드헷) 등이 있으며, 오픈
소스로는 Geronimo(아파치재단), GlassFish(오라클), JBoss(오픈 프로젝트)가 이에 해당합니다

Java EE 사양 중에서 Servlet과 JSP 기술만 구현한 서버를 '서블릿 컨테이너'라 합니다
Tomcat(아파치재단), Resin(Cauho), Jetty(오픈 프로젝트)가 이에 해당합니다

서블릿 라이버러리는 서블릿을 좀더 편리하게 개발할 수 있도록 javax.servlet,GenericServelt
이라는 추상 클래스를 제공합니다. GenericServlet 추상클래스는 Servlet 인터페이스에 선언된
메서드 중에서 service()를 제외한 나머지 메서드를 모두 구현하였습니다. 따라서
서블릿을 만들 때 Servlet 인터페이스를 직접 구현하는 것보다 GenericServelt 클래스를
상속받으면 좀더 쉽고 편리하게 개발할 수 있습니다. 즉 (service) 구현에만 집중하면 되기 때문입니다.
